#### 状态码

```
1xx: 表示目前是协议处理的中间状态，还需要后续操作。
2xx: 表示成功状态。
3xx: 重定向状态，资源位置发生变动，需要重新请求。
4xx: 请求报文有误。
5xx: 服务器端发生错误。
```

```
100：continue 该状态码说明服务器收到了请求的初始部分，并且请客户端继续发送。在服务器发送了 100 Continue 状态码之后，如果收到客户端的请求，则必须进行响应。
例如上传大型文件，先询问服务器是否接受，若同意接受则返回100，客户端发送文件

101：Switching Protocols 在HTTP升级为WebSocket的时候，如果服务器同意变更，就会发送状态码101。

200：OK 服务器成功接受请求，并成功响应

204：No Content 同200但是没有数据，浏览器不用刷新页面.也不用导向新的页面，对于a标签，如果链接的页面响应码为204，页面也不会发生跳转。

206：Partial Content 表示部分内容，它的使用场景为 HTTP 分块下载和断点续传，当然也会带上相应的响应头字段Content-Range。点击下载某个文件，但是需要用户的同意这时候状态码为206

301：Moved Permanently 永久重定向 你的网站从 HTTP 升级到了 HTTPS 了，以前的站点再也不用了，应当返回301，这个时候浏览器默认会做缓存优化，在第二次访问的时候自动访问重定向的那个地址。

302：Moved Temporarily 该资源原本确实存在，但已经被临时改变了位置

304：Last Modified 当协商缓存命中时会返回这个状态码 如果请求中包含 If Modified Since，就说明已经有缓存在客户端。服务端只要判断这个时间和当前请求的文件的修改时间就可以确定是返回 304 还是 200 。

400：Bad Request 语义有误，当前请求无法被服务器理解。除非进行修改，否则客户端不应该重复提交这个请求。请求参数有误。

403：Forbidden 服务器禁止访问

404：Not Found 资源未找到

405：Method Not Allowed 请求方法不被服务器端允许。
```
